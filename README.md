## What It Does

Vikramaditya is an autonomous VAPT tool built for professional security consultants. You give it a target вҖ” a domain, a single IP, or an entire subnet. It runs the full assessment pipeline and produces a submission-ready report.

| Stage | What happens |
|:------|:-------------|
| рҹ”ӯ **Recon** | Subdomain enumeration, DNS resolution, live host discovery, URL crawling, JS analysis, secret extraction |
| рҹ”¬ **Fingerprint** | Tech stack detection (httpx), CVE risk scoring, priority host ranking |
| рҹ”Қ **Scan** | SQLi, XSS, SSTI, RCE, file upload, CORS, JWT, cloud misconfigs, framework exposure |
| рҹ’Ҙ **Exploit** | CMS exploit chains (Drupal, WordPress), Spring actuators, exposed admin panels |
| рҹ§  **Analyze** | AI-powered triage вҖ” finds chains, ranks by impact, kills noise |
| рҹ“Ӣ **Report** | Burp Suite-style HTML report: executive summary, CVSS scores, PoC evidence, remediation |

## Autonomous Agent Mode

The `--autonomous` flag activates the ReAct agent (`agent.py`) which drives the entire assessment without manual intervention вҖ” planning, choosing tools, analysing results, and pivoting to the next attack surface on its own.

```bash
# Autonomous hunt with a 4-hour budget
python3 hunt.py --target example.com --autonomous --time 4

# Watch live decisions as they happen
tail -f recon/example.com/sessions/<session_id>/agent_trace.jsonl

# Inject operator guidance mid-run without stopping the agent
python3 agent.py --bump recon/example.com/sessions/<session_id>/ \
    "Focus on /api/v2/ endpoints вҖ” de-prioritize static assets"
```

The agent operates in a tight loop: **Observe вҶ’ Think (LLM) вҶ’ Act (tool) вҶ’ Observe**. Every decision is logged to `agent_trace.jsonl` for post-engagement review.

The recon pipeline uses the [Chaos](https://chaos.projectdiscovery.io) dataset from ProjectDiscovery вҖ” millions of pre-enumerated subdomains indexed per domain. Free key.

export CHAOS_API_KEY="your-key-here"

`recon.sh` detects `$CHAOS_API_KEY` and auto-injects it into subfinder's provider config on first run. **The key is never stored in any file in this repo.**

## Autopilot API VAPT Engine

The `autopilot_api_hunt.py` module is a **brain-supervised dynamic VAPT engine** for modern SPA + REST API applications. It runs 12 attack phases autonomously with an AI supervisor that decides what to test next based on discoveries.

### How the Brain Supervisor Works

```
Discover endpoints (JS bundle + Django debug + crawl)
  вҶ’ Brain categorizes endpoints and creates prioritized test plan
  вҶ’ Loop:
      Execute top-priority phase
      вҶ’ Brain reviews findings
      вҶ’ Brain decides: CONTINUE / INJECT / SKIP / PIVOT
      вҶ’ Modify test queue based on decision
  вҶ’ Chain building вҶ’ Brain validation (FP removal) вҶ’ Report
```

### Attack Phases

| Phase | Tests |
|-------|-------|
| Auth Bypass | No token, expired, tampered, alg=none |
| IDOR | Sequential ID enum, Base64 decode, PII detection |
| Privilege Escalation | Learner вҶ’ admin endpoint access |
| Business Logic | Score manipulation, workflow bypass, negative values |
| File Upload | Double extension, MIME mismatch, polyglot, S3 arbitrary type |
| Injection | SQLi (time + error), SSTI, command injection |
| Info Disclosure | Django DEBUG, server headers, SMTP creds, AWS keys, DB schema |
| Rate Limiting | Rapid login, password change, reset |
| Token Security | JWT lifetime, refresh token abuse |
| Timing Oracles | User enumeration via response time |
| Chain Building | Cross-reference findings for escalation |
| Brain Validation | FP removal + severity correction via LLM |

### Brain Model Stack

| Role | Model | Speed | Use |
